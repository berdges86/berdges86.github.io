Предварительная обработка данных

Для начала надо вспомнить структуру дашборда из первого урока. В файле было три листа: Now, All и Dashboard:

На листе Now — данные о доходах и расходах только за текущий месяц. В качестве текущего — сентябрь 2021 года.

На листе All — данные о расходах и доходах за все предыдущие месяцы. В таблице — июнь-август 2021 года. Предполагалось, что по окончании текущего месяца данные с листа Now перемещаются на лист All. Таблицы идентичны по структуре, поэтому проблем при переносе возникать не должно.

На листе Dashboard — аналитика по текущему месяцу.

Чтобы построить график с динамикой доходов и две диаграммы сравнения, нужно добавить к исходной таблице три листа:
  1. Days, где доходы за все предыдущие и текущий месяцы суммируются по календарным дням. Данные из этого листа нужны, чтобы построить график.
  2. Weekdays, где доходы за прошедшие месяцы суммируются по дням недели. На этом же листе будут средняя, максимальная и минимальная сумма для каждого дня недели. Weekdays выполняет функцию хранилища данных, которые нужно извлечь на лист For.
  3. For, где в зависимости от даты, указанной на листе Dashboard, выводятся:
    • данные со статистикой по этому дню недели — средняя, максимальная и минимальная сумма доходов;
    • данные о сумме доходов с первого числа месяца по этот день — по текущему и предыдущему месяцам, а также среднее.
Подготовка
  1. Откройте файл. Посмотрите, с помощью каких формул сконструированы эти три новых листа. Внимание: файл пока не нужно копировать.
  2. Ответьте на несколько вопросов по этому файлу.
Какая функция выводит на лист For данные со статистикой сумм доходов по дням недели для диаграммы №1?
Нужная формула, но она не выводит данные по критерию. Обратите внимание на VLOOKUP в ячейках D3, E3, F3.
В ячейку For!C3 из ячейки Dashboard!I2 попадают данные о том, какой сегодня день недели. А дальше для него выводятся среднее, максимум и минимум с листа Weekdays.
Посмотрите, как на листе For выводятся данные для диаграммы №2. Там используется формула SUMIFS, один из аргументов которой выглядит так:
"<="&$D$8
Дело в том, что если вы напишете "<=$D$8", то формула не поймёт, что $D$8 — это ссылка на ячейку, а не просто текст. Поэтому ссылку нужно вынести за кавычки:"<="$D$8. Но в этом случае необходимо дополнительно сообщить программе, что "<=" и $D$8 — это две части одного и того же аргумента. Для этого мы использовали знак &.
Таким образом, аргумент "<="&$D$8 следует читать так: меньше или равно значению, которое находится в ячейке D8.
В диапазоне H:K на листе Days в ячейках нет значений, но есть формулы. Что это за формула, если при отсутствии нужных данных на листе All ячейка пустая, а если данные есть — считается сумма?
Верно! Чтобы программа оставила ячейку пустой, нужно пропустить второй аргумент в записи IF. Поэтому в середине формул стоят два символа ; подряд. Напомним, что у IF три аргумента: условие (1), что сделать, если истина (2), что сделать, если ложь (3).
Идентично настроены формулы для подсчётов сумм по текущему месяцу (столбец B, лист Days) — если данных за конкретные дни нет, ячейки остаются пустыми.
Благодаря формуле IF среднее за 31-й календарный день корректно посчитано: если в месяце 30 дней, эта ячейка для суммы продаж остаётся пустой. В противном случае там бы стояли нули и среднее равнялось не 157,505, а 52,501.
Если на листе All появятся данные ещё за одну неделю, то на листе Weekdays автоматически появится её номер. С помощью какой функции реализована эта механика?
Нужна формула, которая просто выводит все уникальные значения из определённого столбца. IF для этой задачи не подходит. Посмотрите, как мы применили UNIQUE в ячейке A5 на листе Weekdays.
С помощью формулы UNIQUE в столбец A на листе Weekdays выводятся уникальные значения номеров недели с листа All.
И последнее о предварительной обработке данных:
  • На листе Weekdays в ячейке K1 находится формула =TRANSPOSE(A1:H3), которой не было в курсе. Она нужна, чтобы менять расположение данных в строках и столбцах.
  • Суммировать доходы по месяцам и дням, номерам и дням недели, на листах All и Now помогают функции даты и времени: MONTH, WEEKNUM, DAY и WEEKDAY. WEEKNUM(НОМНЕДЕЛИ) выводит номер недели, WEEKDAY(ДЕНЬНЕД) — день недели. Почитать про эти функции можно в справке.
